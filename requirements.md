# 数独游戏 - 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
Sudoku Master（数独大师）

### 1.2 项目目标
开发一款基于 HTML5 的数独游戏，支持多种数独变体玩法，具备精美的 UI 设计和流畅的交互体验。游戏采用纯前端技术栈开发，后续可通过 Capacitor/Cordova 等框架打包为 Android APK 发布。

### 1.3 目标用户
- 数独初学者（通过迷你数独入门）
- 休闲玩家（日常消遣）
- 数独爱好者（挑战高难度和变体玩法）
- 儿童用户（4×4 迷你数独）

### 1.4 技术栈
| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | 原生 HTML5 + CSS3 + JavaScript | 无框架依赖，轻量高效，便于打包 |
| 样式方案 | CSS Variables + Flexbox/Grid | 响应式设计，支持主题切换 |
| 数据存储 | localStorage | 本地存储游戏进度和设置 |
| 打包方案 | Capacitor | 将 Web 应用打包为 Android APK |
| 字体 | Google Fonts (Noto Sans SC) | 中文友好字体 |

## 2. 游戏模式

### 2.1 经典数独（Classic Sudoku）

**棋盘规格：** 9×9 方格，9 个 3×3 九宫格

**规则：**
- 每一行包含 1-9，不重复
- 每一列包含 1-9，不重复
- 每个 3×3 九宫格包含 1-9，不重复

**难度分级（6级）：**

| 难度 | 预填数字数量 | 说明 |
|------|-------------|------|
| 快速 | 45-50 | 入门级，逻辑简单 |
| 简单 | 36-44 | 基础逻辑推理 |
| 中等 | 30-35 | 需要一定技巧 |
| 困难 | 25-29 | 需要高级技巧 |
| 专家 | 22-24 | 需要多种高级技巧 |
| 大师 | 17-21 | 极限挑战，最少17个提示数 |

### 2.2 迷你数独（Mini Sudoku）

#### 2.2.1 6×6 迷你数独
- **棋盘规格：** 6×6 方格，6 个 3×2 宫格
- **规则：** 每行、每列、每个 3×2 宫格包含 1-6，不重复
- **适用人群：** 少年玩家和初学者

#### 2.2.2 4×4 迷你数独
- **棋盘规格：** 4×4 方格，4 个 2×2 宫格
- **规则：** 每行、每列、每个 2×2 宫格包含 1-4，不重复
- **适用人群：** 儿童

### 2.3 杀手数独（Killer Sudoku）

**棋盘规格：** 9×9 方格，9 个 3×3 九宫格 + 不规则"区"

**规则：**
- 遵循经典数独全部规则（行、列、宫不重复）
- 额外增加"区"（cage）：以虚线或颜色划分的不规则区域
- 每个区标注数字，表示该区内所有数字之和
- 区内数字不能重复
- 起始盘面无任何预填数字，完全依靠区的和来推理

**难度分级：** 简单 / 中等 / 困难 / 专家

### 2.4 拼图数独（Jigsaw Sudoku）

**棋盘规格：** 9×9 方格，9 个不规则形状的宫格

**规则：**
- 每行、每列包含 1-9，不重复
- 每个不规则宫格包含 1-9，不重复
- 宫格由不规则线段划分，而非标准 3×3

**难度分级：** 简单 / 中等 / 困难 / 专家

### 2.5 巨无霸数独（Giant Sudoku）

**棋盘规格：** 16×16 方格，16 个 4×4 宫格

**规则：**
- 每行、每列包含 1-16（或 A-F + 1-9 + 0），不重复
- 每个 4×4 宫格包含 1-16，不重复

**难度分级：** 简单 / 中等 / 困难

### 2.6 环状数独（Circular Sudoku）

**棋盘规格：** 圆形，5 个同心环，每环分 10 段

**规则：**
- 每环包含 0-9，不重复
- 每条径向列（从内到外）包含 0-9，不重复

**难度分级：** 简单 / 中等 / 困难

## 3. 功能需求

### 3.1 核心游戏功能

#### 3.1.1 谜题生成与求解
- **生成算法：** 基于回溯法生成完整数独解，再根据难度挖空
- **唯一解保证：** 确保每个谜题有且仅有一个解
- **杀手数独生成：** 先生成完整解，再划分区域并计算和值
- **拼图数独生成：** 预设多种不规则宫格模板，随机选择
- **离线可用：** 所有谜题在本地生成，无需网络

#### 3.1.2 数字输入
- **数字面板：** 屏幕底部显示 1-9（或对应范围）数字按钮
- **点击填数：** 选中格子 → 点击数字按钮填入
- **键盘输入：** 支持物理键盘数字键输入（PC端）
- **长按清除：** 长按已填数字可清除
- **笔记模式：** 切换为笔记模式后，可在格子中标记候选数字（小字显示）

#### 3.1.3 游戏辅助功能

| 功能 | 描述 |
|------|------|
| 撤销 | 撤销上一步操作，支持多步撤销 |
| 重做 | 恢复已撤销的操作 |
| 擦除 | 清除选中格子的数字或笔记 |
| 提示 | 自动填入正确数字（有限次数） |
| 笔记 | 切换笔记模式，标记候选数字 |
| 自动检查 | 开启后实时标记错误数字（红色高亮） |
| 高亮重复 | 高亮显示同行/列/宫中的重复数字 |
| 高亮相同数字 | 选中某数字时，高亮棋盘上所有相同数字 |
| 高亮关联区域 | 选中格子时，高亮同行/列/宫 |

#### 3.1.4 游戏状态
- **计时器：** 记录游戏用时，支持暂停
- **错误计数：** 记录错误次数（最多3次，3次错误游戏结束）
- **完成度：** 显示已填数字占总空格数的百分比
- **最佳记录：** 记录每个难度级别的最佳用时

### 3.2 每日挑战
- 每天生成一个固定谜题（基于日期种子）
- 所有玩家同一天面对同一谜题
- 显示完成每日挑战的统计
- 每日挑战支持经典数独和杀手数独

### 3.3 存档系统
- **自动保存：** 每次操作后自动保存当前进度
- **多存档槽：** 支持最多 5 个未完成的游戏存档
- **继续游戏：** 主页显示未完成的游戏，可继续
- **存档信息：** 显示难度、完成度、用时

### 3.4 统计系统
- 游戏完成数（按模式/难度分类）
- 最佳用时（按模式/难度分类）
- 平均用时
- 连胜天数
- 当前胜率
- 总游戏时长

## 5. 各模式特殊 UI 需求

### 5.1 杀手数独
- 区的边界用虚线或不同底色区分
- 每个区左上角显示和值数字（小字斜体）
- 区内数字不能重复的规则需在 UI 中体现
- 无预填数字，起始盘面全空

### 5.2 拼图数独
- 不规则宫格用不同底色区分（9种柔和色彩）
- 宫格边界用粗线标示
- 提供宫格形状预览

### 5.3 巨无霸数独
- 使用 1-9 + A-G 表示 16 个数字
- 格子尺寸适当缩小
- 支持缩放和拖拽浏览
- 数字面板显示 1-9 + A-G

### 5.4 环状数独
- 圆形棋盘渲染（Canvas/SVG）
- 5 个同心环，每环 10 段
- 径向和环向高亮
- 支持触摸选中扇形格子

### 5.5 迷你数独
- 6×6：数字面板显示 1-6
- 4×4：数字面板显示 1-4
- 格子尺寸放大，适合儿童操作
- 色彩更鲜艳活泼

## 8. APK 打包方案

### 8.1 打包框架
使用 **Capacitor**（推荐）将 Web 应用打包为 Android APK：
- Capacitor 是 Ionic 团队开发的跨平台打包工具
- 比 Cordova 更现代，性能更好
- 支持原生 API 访问（如振动反馈）

### 8.2 打包流程
1. 完成 Web 应用开发
2. `npm install @capacitor/core @capacitor/cli`
3. `npx cap init`
4. `npx cap add android`
5. 将 Web 资源复制到 Android 项目
6. 使用 Android Studio 编译 APK

### 8.3 原生功能集成
- **振动反馈：** 填数/错误时轻微振动
- **状态栏：** 适配深色/浅色主题
- **全屏模式：** 隐藏系统导航栏
- **应用图标：** 自定义启动图标
